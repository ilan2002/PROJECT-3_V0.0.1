EMBEDDED C CODING STANDARDS
----------------------------
  1. MISRA-C 2012 compliance where applicable.
  2. No dynamic memory allocation (no malloc / calloc / free).
  3. All functions must have a single entry and single exit point.
  4. Maximum function length: 50 lines of executable code.
  5. Every function must have a Doxygen-style comment block.
  6. All global variables must be declared static in main.c.
  7. Use fixed-width integer types from stdint.h.
  8. Use ifndef include guards in main.h.
  9. All #define macros must be UPPER_SNAKE_CASE.
  10. All function names must be lower_snake_case.
  11. Struct and typedef names must be PascalCase (e.g. SensorData_t).
  12. Error codes must use an enum, not magic numbers.
  13. Hardware register accesses must use volatile pointers or CMSIS macros.
  14. ISR functions must be prefixed with ISR_ and kept minimal.
  15. Avoid recursion entirely.

OUTPUT FORMAT
-------------
  1. Output ONLY valid Embedded C code.
  2. Wrap main.h additions in: /* -- ComponentID: Title -- */
  3. Wrap main.c additions in: /* -- ComponentID: Title -- */
  4. Never output code for files other than main.c and main.h.
  5. Tag every function: @trace DD-COMP-XXX SW-REQ-XXX
